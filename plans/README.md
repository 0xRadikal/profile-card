## Findings considered and rejected

- **History ArrowUp/ArrowDown off-by-one**: traced with a 3-entry history;
  navigation is correct (idx boundaries hold, ArrowDown lands on '' at the
  end). Not a bug. (So nobody re-audits it.)
- **`cat <injection>` XSS via arg**: `files.get(arg)` returns undefined for
  any unknown key, so the `printHTML` branch is only reached for the fixed,
  trusted key set; unknown args fall to the `printText` (textContent) branch.
  Not exploitable.
- **`contactForm = qs('#contact form.card-form')` binds nothing**: verified a
  `<form class="card-form">` exists inside `#contact` (index.html:359). Binds
  correctly. Not a bug.
- **All `printHTML` callers**: every dynamic value comes from the static
  in-file data arrays (blogs/skillset/socials/milestones/achievements) or
  library-controlled strings — no untrusted input reaches innerHTML after the
  fetch fix. No further XSS surface in `index.js`.
